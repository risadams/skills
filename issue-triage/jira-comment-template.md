# Jira Comment Template — Issue Triage

Use this template to render the Phase 6 triage report as a Jira comment in **Jira wiki markup** (not GitHub-flavored markdown). Replace every `{placeholder}` with content from the report. Keep the AI disclaimer panel at the bottom verbatim.

---

## Format reference (markdown → Jira wiki markup)

| Markdown | Jira wiki markup |
| :--- | :--- |
| `# H1` | `h1. H1` |
| `## H2` | `h2. H2` |
| `### H3` | `h3. H3` |
| `**bold**` | `*bold*` |
| `_italic_` | `_italic_` |
| `` `inline code` `` | `{{inline code}}` |
| ``` ```block``` ``` | `{code}block{code}` (or `{code:java}` etc.) |
| `- item` | `* item` |
| `1. item` | `# item` |
| `> quote` | `{quote}quote{quote}` |
| `[text](url)` | `[text\|url]` |
| Horizontal rule | `----` |
| Table row `\| a \| b \|` | `\| a \| b \|` (header row uses `\|\|` delimiters) |

Panels (used in this template):

```
{panel:title=Title|borderColor=#ccc|bgColor=#f4f5f7}
content
{panel}

{info}info content{info}
{note}note content{note}
{warning}warning content{warning}
```

---

## Notes for the rendering agent

1. **Strip empty hypothesis blocks.** If the council produced fewer than 3 hypotheses, omit the unused `h4.` sections rather than leaving placeholder text.
2. **Trim "Open questions" and "Signals consulted"** to lines that have actual content. An empty `* ` bullet looks broken in Jira.
3. **Keep the disclaimer panel exactly as shown** — same title, colors, and wording. The colors map to Atlassian's standard "warning" palette so the panel reads consistently across Jira themes.
4. **Don't include any preamble** ("Hi team, here's a triage…") or sign-off ("Let me know if…"). The comment should start with the `h2.` heading and end with the disclaimer panel.
5. **Escape pipes inside table cells** as `\|` if any hypothesis or signal text contains a literal `|`.
6. **Code samples** (stack frames, error messages, snippets) belong inside `{code}...{code}` blocks for readability.
